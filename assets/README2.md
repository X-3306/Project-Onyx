# Runtime Assets

This directory intentionally excludes private generated artifacts. Before a
release build, generate or place these files here.

## `model.onnx`

Generated by `build.py`. The file is based on the bundled SqueezeNet 1.0 ONNX
image-classification model from Hugging Face's ONNX Model Zoo mirror.

- source model: `assets/base/squeezenet1.0-12.onnx`
- input: `data_0`, `float[1, 3, 224, 224]`
- output: `softmaxout_1`, `float[1, 1000, 1, 1]`

The graph is used as a real ONNX Runtime image-classification workload. Demo key
material is not returned by inference. `build.py` writes two synchronized vaults
into the ONNX file:

1. a hidden authenticated weight vault embedded in the LSBs of `float32`
   initializer bytes;
2. a metadata fallback vault retained for compatibility and easier build-time
   inspection.

The metadata fallback contains:

- `schema`
- `kdf`
- `kdf_iterations`
- `kdf_salt`
- `trigger_hmac`
- `vault_iv`
- `vault_ct`
- `vault_hmac`

At runtime, the C++ host first extracts the weight vault from model bytes in the
PE resource section. It derives keys from the environment fingerprint, verifies
the trigger and vault MACs, and decrypts the vault to recover the 32-character
demo key material. Metadata fallback is used for older generated assets.

## `license_module.wasm.aes`

Generated by `build.py --wasm-input <compiled.wasm>`. The expected format is:

1. first 16 bytes: AES-CBC IV;
2. remaining bytes: PKCS#7 padded ciphertext of the compiled Wasm module.

The host derives the AES-256 key by applying SHA-256 to the 32-character key
material unlocked from the ONNX weight vault or metadata fallback.

## `downlink_update.onnx`

Generated by `build.py downlink-build` for lab validation. The update model is
derived from the reference model with seeded fine-tuning-like weight noise unless
an operator provides a real fine-tuned cover model through `--cover-model`.

The runtime never executes this update model. It parses the update bytes,
compares the update weights against the embedded reference model, selects only
weights whose absolute delta is at least `1e-5`, and attempts to extract an
authenticated heartbeat-only directive from those candidate LSBs.
