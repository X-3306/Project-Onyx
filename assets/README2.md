# Runtime Assets

This directory intentionally excludes private generated artifacts. Before a
release build, generate or place these files here.

## `model.onnx`

Generated by `build.py`. The file is a small valid ONNX graph with two inputs:

- `input_ids`: `int64`, shape `[1, 16]`
- `attention_mask`: `int64`, shape `[1, 16]`

The graph is used as a real ONNX Runtime workload. Demo key material is not
returned by inference. Instead, `build.py` writes an authenticated metadata vault
into the ONNX file:

- `schema`
- `kdf`
- `kdf_iterations`
- `kdf_salt`
- `trigger_hmac`
- `vault_iv`
- `vault_ct`
- `vault_hmac`

At runtime, the C++ host derives keys from the environment fingerprint, verifies
the trigger and vault MACs, and decrypts the vault to recover the 32-character
demo key material.

## `license_module.wasm.aes`

Generated by `build.py --wasm-input <compiled.wasm>`. The expected format is:

1. first 16 bytes: AES-CBC IV;
2. remaining bytes: PKCS#7 padded ciphertext of the compiled Wasm module.

The host derives the AES-256 key by applying SHA-256 to the 32-character key
material unlocked from the ONNX metadata vault.
